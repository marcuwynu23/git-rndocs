# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| latest | Full support |
| older | Community support only |

## Reporting a Vulnerability

If you discover a security vulnerability, **do not open a public issue**. Please report it privately.

### How to Report

- Email the maintainers directly (see commit history for contact)
- Alternatively, use GitHub's private vulnerability reporting at:
  https://github.com/marcuwynu23/git-rndocs/security/advisories

### What to Include

- Type of vulnerability
- Steps to reproduce
- Affected versions
- Impact description
- Any suggested fix (optional)

### Response Timeline

- **Acknowledgement** within 48 hours
- **Status update** within 5 business days
- **Fix released** within 30 days for critical issues

## Security Best Practices for Users

- Always use the latest release
- Verify checksums before installing binaries
- Review release notes for security-related changes
- Run with least privilege — the tool only needs read access to your Git repository

## Disclosure Policy

We follow coordinated disclosure:

1. Vulnerability reported privately
2. Fix developed and tested
3. Patch released with advisory
4. Public disclosure 7 days after release
